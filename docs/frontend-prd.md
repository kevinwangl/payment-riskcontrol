## 一、产品概述

### 1.1 产品定位

面向收单支付 SaaS 平台的多租户风控操作台，服务于 Platform 运营团队、ISO 管理员、Partner/Merchant 管理员，覆盖事前准入、事中决策、事后监控的完整风控业务流程。

### 1.2 设计原则

- 效率优先：审核员日均处理数百笔 REVIEW 交易，交互路径最短化
- 数据驱动：所有决策有数据支撑，关键指标实时可见
- 多租户隔离：不同租户只能看到和操作自己权限范围内的数据
- 渐进式复杂度：常用操作一步到位，高级功能按需展开

### 1.3 登录页 `/login`

**布局：** 全屏深蓝黑背景，中央登录卡片

- 背景：深色底 + 科技感粒子网格动画（缓慢漂浮连线），左侧或底部可放品牌 Slogan
- 登录卡片（居中，宽 400px）：
  - Logo + 系统名称
  - 用户名输入框
  - 密码输入框
  - 登录按钮（科技蓝，hover 发光）
  - 忘记密码链接
- 登录成功：按钮 loading → 卡片向上淡出 → 跳转看板页
- 登录失败：输入框边框闪红 + 错误提示文案

## 二、用户角色与权限模型

### 2.1 角色定义

| 角色 | 所属层级 | 职责 |
|------|---------|------|
| Platform Admin | Platform | 全局规则管理、系统配置、所有租户数据可见 |
| Platform Risk Analyst | Platform | 全局风控分析、模型治理、合规报告 |
| ISO Admin | ISO | 管理旗下 Partner/Merchant 的规则和名单 |
| ISO Review Agent | ISO | 审核旗下商户的 REVIEW 交易和案件 |
| Merchant Admin | Merchant | 查看自己的风控数据、配置商户级规则 |

| 功能模块 | Platform Admin | Platform Analyst | ISO Admin | ISO Agent | Merchant Admin |

| 规则配置 | 全局 + Platform 规则 | 只读 | ISO 级规则 | — | Merchant 级规则 |
| 黑白名单 | 全局 + 平台级 | 只读 | ISO 级 | — | Merchant 级 |
| 人工审核 | 全部 | 只读 | 本 ISO | 本 ISO（操作） | — |
| 交易查询 | 全部 | 全部 | 本 ISO | 本 ISO | 本商户 |
| Chargeback | 全部 | 只读 | 本 ISO | 本 ISO（操作） | 本商户（只读） |
| 案件管理 | 全部 | 全部 | 本 ISO | 本 ISO（操作） | — |
| 模型治理 | 全部 | 全部（操作） | — | — | — |
| 报表合规 | 全部 | 全部 | 本 ISO | — | 本商户 |
| 系统管理 | 全部 | — | — | — | — |

### 2.3 数据隔离规则

- 所有列表页、详情页、看板数据均按当前用户的 `tenant_type` + `tenant_id` 自动过滤
- Platform 级用户可通过顶部租户切换器查看任意 ISO/Merchant 的数据
- ISO 级用户只能看到自己旗下的 Partner/Merchant 数据
- Merchant 级用户只能看到自己的数据

## 五、模块详细设计

### 模块 1：风控总览看板

**分期：** 一期 MVP

**页面：** `/dashboard`

**目标：** 提供风控系统核心指标的实时概览，帮助管理者快速判断系统健康状态。

**数据来源：** Redshift（聚合指标）+ Redis（实时计数器）

**核心组件：**

1. 指标卡片区（顶部横排）

| 指标 | 数据源 | 刷新频率 |
|------|--------|---------|
| 今日交易量 | Redshift `v_merchant_daily_metrics` | 5 分钟 |
| 放行率 / 拒绝率 / REVIEW 率 | Redshift | 5 分钟 |
| 拒付率（本月） | Redshift `v_merchant_chargeback_rate` | 1 小时 |
| 决策延迟 P99 | CloudWatch / Prometheus | 1 分钟 |

异常状态：拒付率 > 0.9% 或 P99 > 50ms 时，对应卡片标红。

2. 趋势图区

- 近 7 天交易量趋势（折线图，按 APPROVE/DECLINE/REVIEW 分色）
- 近 30 天拒付率趋势（折线图，标注 Visa 0.9% 和 MC 1.5% 阈值线）

3. 商户风险排行（表格）

| 列 | 说明 |
|----|------|
| 商户名称 | 可点击跳转商户详情 |
| 拒付率 | 本月，超标标红 |
| REVIEW 率 | 本月 |
| 风险等级 | LOW/MEDIUM/HIGH 标签 |

ISO 用户只看到自己旗下商户，Merchant 用户不显示此表格。

**页面布局（上下结构）：**

```
┌─────────────────────────────────────────────────────────┐
│  指标卡片区（4 等分横排，高 120px）                         │
│  [今日交易量] [放行/拒绝/REVIEW率] [拒付率] [P99延迟]      │
├────────────────────────────┬────────────────────────────┤
│  交易量趋势图（左 50%）      │  拒付率趋势图（右 50%）      │
│  高 300px                   │  高 300px                   │
├────────────────────────────┴────────────────────────────┤
│  商户风险排行表（全宽，高度自适应）                          │
└─────────────────────────────────────────────────────────┘
```

**分期：** 一期 MVP（规则 CRUD），二期（规则效果分析）

**对应后端：** system-design-v3 第六章（规则引擎）、3.2 节（决策结果结构）

#### 页面 3.1：规则列表 `/rules`

左侧：多租户树（Platform > ISO > Merchant），选择租户后右侧刷新。

右侧规则表格：

| 列 | 说明 |
|----|------|
| 规则 ID | — |
| 规则名称 | — |
| 优先级 | 数字，越小越优先 |
| 适用模式 | CP / CNP / ALL |
| 条件表达式 | 截断显示，hover 展开 |
| 决策 | APPROVE / REVIEW / DECLINE 标签 |
| 建议 | REQUIRE_3DS 等标签 |
| 状态 | 启用 / 禁用 |
| 版本 | 当前版本号 |
| 操作 | 编辑、禁用/启用、查看历史版本 |

Platform 级规则在 ISO/Merchant 视图中灰色只读显示，标注"平台强制规则"。

```

│  (宽 240px)   │  (剩余宽度)                               │
│              │                                          │

│      Merch1  │  │ R1001 | 高额CNP拦截 | DECLINE | 启用│  │
│      Merch2  │  │ R1002 | 境外卡审核   | REVIEW  | 启用│  │
│    ▼ ISO_B   │  │ ...                                │  │
│      ...     │  └────────────────────────────────────┘  │
└──────────────┴──────────────────────────────────────────┘
```

#### 页面 3.2：规则编辑器 `/rules/edit/:id`

分区：
1. 基本信息：规则名称、优先级、适用模式（CP/CNP/ALL）
2. 条件配置：
   - 可视化模式：条件组构建器（字段下拉 + 运算符 + 值输入），支持 AND/OR 组合
   - 代码模式：直接编辑 DSL 表达式（高级用户切换）
3. 动作配置：
   - Decision：APPROVE / REVIEW / DECLINE（单选）
   - Suggestions：REQUIRE_3DS / REQUIRE_PIN / REQUIRE_OTP（多选）
   - Reason Code：下拉或自定义输入
4. 沙箱测试：输入模拟交易 JSON，实时显示规则是否命中

操作流程：保存草稿 → 沙箱测试 → 提交审批 → 审批通过后发布

#### 页面 3.3：规则效果分析 `/rules/analytics`（二期）

- 每条规则的命中次数、命中率趋势
- 规则导致的 DECLINE 中，后续是否产生 Chargeback（误杀率分析）
- 数据来源：Redshift `risk_decision_log`

### 模块 5：人工审核工作台

**分期：** 一期 MVP

**对应后端：** system-design-v3 3.1 节（REVIEW 处置）、数据库 `review_tasks` 表

**页面：** `/review`

**目标：** 审核员高效处理 REVIEW 交易，支持键盘快捷操作。

**布局：** 左右分栏

左栏（交易上下文）：
- 交易基本信息：金额、时间、MCC、CP/CNP
- 卡片信息：BIN、发卡国、AVS/CVV 结果
- 风险评分：分数 + 命中规则标签（如 `[R1001: 高额境外CNP - REVIEW]`）
- 持卡人行为：Velocity 数据（同卡近 1h 交易次数/金额）
- 关联分析：同设备/同 IP 关联的其他卡数量
- 商户信息：商户名称、风险等级、历史拒付率

右栏（操作区）：
- 判定按钮：正常（LEGIT）/ 欺诈（FRAUD）/ 可疑（SUSPICIOUS）
- 欺诈类型下拉（当选择 FRAUD 时展开）：盗卡、账户盗用、友好欺诈等
- 批注输入框
- 提交

**快捷键：**
- `L`：标记 LEGIT 并提交，自动切换下一条
- `F`：标记 FRAUD（需选择欺诈类型后回车确认）
- `S`：标记 SUSPICIOUS 并提交
- `→`：跳过当前，查看下一条

**队列管理：**
- 顶部显示：待审核总数、今日已审核数、平均处理时间
- 排序：默认按风险评分降序（高风险优先）
- 筛选：按 ISO、商户、风险评分区间

**数据推送：** 新 REVIEW 交易通过 WebSocket 实时推送，队列自动更新。

**页面布局（左右分栏）：**

```
┌──────────────────────────────────────────────────────────────┐
│  队列状态栏（全宽，高 48px）                                    │
│  待审核: 23  |  今日已审: 156  |  平均处理: 18s  |  筛选 ▼     │
├──────────────────────────────────┬───────────────────────────┤
│  左栏 - 交易上下文（60%）          │  右栏 - 操作区（40%）       │
│                                  │                           │
│  ┌─ 交易信息 ──────────────┐     │  风险评分环形图             │
│  │ $3,200 | CNP | 5812餐饮 │     │  ┌───┐                    │
│  └─────────────────────────┘     │  │ 67 │ (黄色)            │
│  ┌─ 卡片信息 ──────────────┐     │  └───┘                    │
│  │ BIN:4111 | US | AVS:Y   │     │                           │
│  └─────────────────────────┘     │  命中规则标签               │
│  ┌─ Velocity ──────────────┐     │  [R1001] [R2003]          │
│  │ 同卡1h: 5笔/$8,200      │     │                           │
│  └─────────────────────────┘     │  ── 判定 ──               │
│  ┌─ 关联分析 ──────────────┐     │  [LEGIT(L)] [FRAUD(F)]    │
│  │ 同设备3卡 | 同IP 1卡     │     │  [SUSPICIOUS(S)]          │
│  └─────────────────────────┘     │                           │
│  ┌─ 商户信息 ──────────────┐     │  欺诈类型（FRAUD时展开）    │
│  │ 风险:MEDIUM | 拒付:0.6%  │     │  [下拉选择]               │
│  └─────────────────────────┘     │  批注: [________]         │
│                                  │  [提交]                   │
├──────────────────────────────────┴───────────────────────────┤
│  快捷键提示栏: L=正常  F=欺诈  S=可疑  →=跳过                   │
└──────────────────────────────────────────────────────────────┘
```

## 六、全局交互规范

### 6.1 视觉风格：AI 科技感

整体采用深色主题 + 科技蓝色调，营造智能风控指挥中心的视觉氛围。

**色彩体系：**
- 主背景：深蓝黑 `#0A1628`，卡片/面板背景：`#111D2E`
- 主色调：科技蓝 `#1890FF`，辅助色：青色 `#00D8FF`
- 风险色阶：低风险绿 `#52C41A` → 中风险黄 `#FAAD14` → 高风险红 `#FF4D4F`
- 文字：主文字 `#E8ECF1`，次文字 `#8C99A8`

**全局动效：**
- 数据加载：数字跳动计数器（CountUp），指标卡片数值变化时数字滚动过渡
- 页面切换：内容区淡入 + 微上移（fade-in + translateY），时长 300ms
- 图表渲染：折线图/柱状图数据点依次绘入（ECharts animation），营造数据"流入"感
- 卡片 hover：微发光边框（box-shadow 扩散科技蓝光晕），轻微上浮 2px

### 6.2 模块专属动效

**风控总览看板：**
- 指标卡片：异常指标（拒付率超标、P99 超限）触发边框脉冲呼吸光效（蓝→红渐变，2s 周期）
- 拒付率趋势图：阈值线（0.9%/1.5%）使用虚线 + 微闪烁，接近阈值的数据点放大高亮
- 商户风险排行：新增高风险商户时，行项从左侧滑入 + 短暂高亮闪烁

**人工审核工作台：**
- 审核提交后：当前卡片向左滑出 + 透明度渐隐，下一条从右侧滑入，无整页刷新
- 判定按钮：LEGIT 点击后绿色脉冲扩散，FRAUD 点击后红色脉冲扩散，提供即时操作反馈
- 风险评分：环形进度条动态填充，颜色随分值从绿渐变到红

**规则编辑器：**
- 条件组构建：新增条件块时，块从上方下落 + 弹性回弹（spring easing）
- 沙箱测试：命中时条件块边框闪绿光，未命中灰化，直观展示规则匹配路径
- 规则发布：进度条使用粒子流动效果（科技蓝粒子沿进度条方向流动）

**模型治理：**
- 模型部署进度：EKS Pod 同步状态用节点拓扑图展示，每个节点完成时亮起蓝光
- SHAP 特征归因：瀑布图各特征条从零点向左/右动态伸展，正贡献蓝色、负贡献红色
- Shadow 对比：双模型评分分布直方图叠加渲染，切换时柱体高度平滑过渡

**交易详情：**
- 风控决策路径：用流程图动画展示交易经过的每个引擎节点（黑名单→Velocity→规则→模型），命中节点亮起，未命中灰化
- 关联分析：设备/IP/卡号关联关系用力导向图（force-directed graph）展示，节点间连线有粒子流动

### 6.3 列表页通用规范

- 默认分页 20 条/页
- 支持列排序（点击表头）
- 筛选条件区默认折叠，点击"筛选"展开
- 空状态：深色背景 + 科技风线条插图 + 引导文案
- 加载状态：骨架屏（Skeleton）+ 微光扫描效果（shimmer）

### 6.4 空状态与异常状态

**各模块空状态文案：**

| 页面 | 空状态文案 |
|------|-----------|
| 审核工作台 | 🎉 所有审核任务已处理完毕（显示最近 1h 审核统计） |
| 规则列表 | 当前租户暂无规则，点击"新建规则"开始配置 |
| 黑白名单 | 暂无名单数据 |
| 交易查询 | 未找到匹配的交易记录，请调整筛选条件 |
| Chargeback 列表 | 暂无 Chargeback 记录 |
| 案件列表 | 暂无案件 |
| 商户入网审核 | 暂无待审核的入网申请 |

**异常状态：**

| 场景 | 处理 |
|------|------|
| 无权限访问 | 显示 403 页面："您没有权限访问此页面，请联系管理员" |
| 页面不存在 | 显示 404 页面："页面不存在"+ 返回首页按钮 |
| 接口超时/错误 | 页面内 inline 错误提示："数据加载失败，点击重试"+ 重试按钮 |
| WebSocket 断连 | 审核工作台顶部显示黄色提示条："实时连接已断开，正在重连..." |
| 降级模式 | 看板页顶部显示橙色提示条："风控引擎当前处于降级模式"（读取 `degraded` 标记） |

### 6.4 操作确认

- 高危操作（规则发布、模型部署、名单批量导入、商户冻结）需二次确认弹窗
- 确认弹窗显示影响范围（如"此规则将影响 ISO_2001 下 128 个商户"）
- 弹窗出现时背景模糊（backdrop-filter: blur）+ 弹窗从中心缩放弹出

### 6.5 实时通知

- WebSocket 推送场景：新 REVIEW 交易、Chargeback 接收、P0 案件创建、模型部署完成
- 通知中心：右上角铃铛，未读数角标（脉冲动效），点击展开通知列表
- P0 告警：全屏顶部滑入红色告警条，伴随边框闪烁，需手动关闭

### 6.6 响应式

- 最小支持宽度 1280px（企业级后台，不需要移动端适配）
- 审核工作台针对 1920px 宽屏优化双栏布局

### 6.7 技术选型

- 框架：React + Ant Design（与 V3 14.1 节一致）
- 图表：ECharts（深色主题 + 自定义科技风配色）
- 动效：Framer Motion（页面切换、卡片动效）
- 关系图：AntV G6（关联分析力导向图）
- 状态管理：React Query（服务端状态）
- 实时通信：WebSocket（审核工作台、告警推送）
